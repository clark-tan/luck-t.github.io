ionic从环境配置到打包

发表于 May 5 2017 | 分类于 Ionic

之前已经说过Ionic 的基本安装以及创建应用了，今天说说怎么配置环境进行打包，里面躺的坑已经无力吐槽了。

安装打包软件

  • 安卓安装 android studio。
  • ios 安装XCode。
  • 下载jdk 这里本人是安装1.8.0版本

Windows环境变量配置

变量名：ANDROID_HOME  变量值C:\Users\clark\AppData\Local\Android\sdk
变量名：JAVA_HOME     变量值C:\Program Files\Java\jdk1.8.0_151

用户变量

Mac环境变量配置

首先再终端输入 sudo vim/etc/profile,按下i，显示insert，进入输入模式（注：在终端输入 /usr/libexec/java_home可以得到JAVA_HOME的路径）

输入配置如下：


按ESC，进入保存
输入：wq！保存

想要马上生效的话，输入 source /etc/profile 运行profile配置

检查环境输入 echo $JAVA_HOME 以及 $adb version 和 $android

安装cnpm

命令行运行npm install -g cnpm --registry=https://registry.npm.taobao.org(需要获取管理员权限在运行如果是mac运行sudo -s获取最高权限)

安装nrm

命令行运行 npm install -g nrm 安装完成后运行 nrm use taobao

安装ionic

运行npm install -g cordova ionic(如果是mac则运行sudo npm install -g cordova ionic)

下载同步工具

下载TortoiseGit或者GitHub Desktop（mac下载scourcetree）然后同步项目到指定文件夹运行项目

调试代码

打开命令行进入项目文件夹，运行npm i，运行ionic serve进入调试模式编写页面代码（这里面有个node-sass的坑，如果报这个错加上--unsafe-perm就可以）

添加平台

添加Android和ios平台 cordova platform add android(ios)

添加所需插件

运行 cordova plugin add pluginName (pluginName就是插件名字，例如cordova-hot-code-push)

添加完插件后运行代码cordova build android（ios）
